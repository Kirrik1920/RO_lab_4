Висновок до Лабораторної роботи №4
Під час виконання цієї лабораторної роботи я реалізував вирішення класичної задачі паралельного програмування — проблеми "Читачів та Письменників" (Readers-Writers problem). Метою було забезпечити коректний одночасний доступ до спільного ресурсу для кількох потоків.

Основні аспекти роботи:

Моделювання: Було створено програму, що запускає кілька паралельних завдань (Task), які імітують поведінку трьох "читачів" та двох "письменників".

Спільний ресурс: Використовувався спільний ресурс у вигляді цілочисельної змінної (resource).

Синхронізація: Для керування доступом до ресурсу та захисту лічильника читачів (readersCount) було використано два об'єкти SemaphoreSlim.

Один семафор (readerSemaphore) виступав у ролі м'ютекса для безпечної зміни лічильника readersCount.

Другий семафор (writerSemaphore) використовувався для забезпечення ексклюзивного доступу для письменників та для блокування письменників, доки хоча б один читач активний.

Обрана в коді схема реалізує пріоритет для читачів. Це означає, що:

Кілька читачів можуть одночасно читати ресурс.

Письменник отримує ексклюзивний доступ (блокуючи і читачів, і інших письменників).

Поки хоча б один читач володіє доступом, жоден письменник не може почати запис. Нові читачі, що прибувають, можуть одразу починати читання, "відсуваючи" письменника в черзі.

У ході роботи я здобув практичні навички застосування семафорів для вирішення поширеної задачі синхронізації. Я зрозумів логіку патерну "Читачі-Письменники" та усвідомив, що обраний підхід (пріоритет читачів) може призвести до "голодування" письменників (writer starvation) у випадку, якщо потік читачів є безперервним.
